> **Status:** reference (일반론)
> **Date:** 2026-06-08

## 0. 척추 원리 — 마케팅을 엔지니어링처럼

이 플레이북 전체를 관통하는 한 가지: **던져놓고 기도(spray & pray)하지 말고, 계측하고 수렴한다.** 마케팅을 "감(感)의 영역"이 아니라 **가설 → 실험 → 측정 → 반복**의 엔지니어링 루프로 다룬다. 못 하는 도메인을 새로 배우는 게 아니라, 이미 가진 엔지니어링 규율을 옮겨오는 것이다.

| 엔지니어링 | 마케팅 |
|---|---|
| 추측으로 고치지 말고 **계측 먼저** | 채널 들이붓기 전에 추적 링크·전환율부터 (§4) |
| 단위 테스트 통과 ≠ 동작 확인 | 설치수 ≠ 성공. **D1/D7 리텐션**이 진짜 통과 기준 (§4) |
| 프로파일링 없는 최적화는 헛발 | 전환 데이터 없는 유료광고는 헛돈 (§1 대전제, §2 Tier3) |
| 가설 → 재현 → 로그로 원인 좁히기 | 가설(포지셔닝) → 출시 → 사용 데이터로 검증 (§5) |
| 복리 자산(좋은 추상화) vs 일회성 핫픽스 | ASO·콘텐츠(복리) vs 런칭 스파이크(일회성) (§2) |

> 이 사고는 새로운 게 아니라 `CLAUDE.md` 응답 원칙("추측으로 행동하지 말고 확인 먼저", "낙관 금지 — 검증 안 된 걸 완료로 선언 말 것", "배제≠규명")을 마케팅 도메인에 적용한 것이다. 아래 모든 섹션은 이 한 원리의 따름정리다.

### 0.1 단, 단계 의존성 — 0→1에선 정성이 주인공, 정량은 규율 장치

"엔지니어링처럼"을 "다 측정해서 결정해라"로 오해하면 안 된다. **초기(0→1) 단계에선 정량의 비중이 원리적으로 작다:**

- **N이 작아 통계가 약하다.** 유저 수십~수백 명에서 나오는 전환율·리텐션은 신뢰구간이 너무 넓어 의사결정 근거로 부적합. A/B 테스트가 의미 있으려면 표본이 수천은 돼야 하고, 그 규모는 한참 뒤다.
- **이 단계의 진짜 신호는 정성이다.** 숫자 10개보다 **유저 한 명의 인터뷰**, 커뮤니티 댓글 하나의 온도, "이거 진짜 쓸만하다" 한 마디가 정보량이 몇 배 크다. 작은 N에선 깊은 정성 관찰이 얕은 정량보다 분산이 작다.
- **결과를 만드는 건 정성.** 메시지가 정확한지, 카피가 사람 마음을 건드리는지, 제품이 약속을 첫 세션에 지키는지 — 이게 안 되면 측정할 *전환 자체*가 없다. **정량은 이미 좋은 정성을 확인·증폭할 뿐, 없는 걸 만들지 못한다.**

따라서 이 단계에서 정량(추적 링크·전환율 보기)의 역할은 *의사결정 근거*가 아니라 **규율 장치(난간)**다 — "설치수에 도취되지 말 것", "N이 커졌을 때 후회 않게 계측은 미리 깔아둘 것" 같은 자기기만 방지용. **측정이 의사결정을 주도하는 건 N이 충분히 커진 다음.** 0→1은 측정의 시간이 아니라 *가설을 날카롭게 벼리는* 시간이다.

## 1. 대전제 — 인디는 "유료광고로 사용자 사는" 게임을 못 이긴다

대형 앱은 LTV(유저 생애가치)가 높아 CAC(획득비용)를 광고로 메꾼다. 무료/저가 인디는 LTV가 낮아 **유료광고 ROI가 거의 항상 마이너스**다. 그래서 인디 정석은:

> **오가닉(공짜 유입) 우선 → 유료는 데이터 쌓인 뒤 보조/실험.**

이 순서를 뒤집으면 — 전환율(설치→사용→잔존) 검증 전에 광고부터 태우면 — **새는 양동이에 물 붓기**가 된다. 돈만 나간다.

## 2. 채널 우선순위

### Tier 1 — 무조건 먼저 (공짜·복리)
- **ASO (App Store Optimization)** — 앱스토어 트래픽의 60~70%가 검색. 제품페이지 *디자인*이 아니라 **이름·부제·키워드 필드**에 검색어를 심었는지가 핵심. 한 번 정비하면 계속 일하는 복리 자산.
- **콘텐츠 마케팅** — 타겟이 모이는 곳에서 "앱 홍보"가 아니라 **유용한 콘텐츠**를 뿌리고 끝에 앱을 둔다. 광고 알레르기를 우회하는 거의 유일한 통로.

### Tier 2 — 출시 모멘텀 (1회성 스파이크)
- **런칭 플랫폼**: Product Hunt, (국내) 디스콰이엇·GeekNews(긱뉴스).
- **커뮤니티**: 타겟이 이미 모인 서브레딧·카페·전공 커뮤니티.
- **피칭**: 관련 분야 뉴스레터·블로거·기자에게 직접 메일.
- 이들은 **출시 주(launch week)에 트래픽 스파이크** → 차트 랭킹 부스트용. 지속 채널이 아님.

### Tier 3 — 유료 (데이터 쌓인 뒤)
- Apple Search Ads(브랜드 키워드 방어부터), 인스타/틱톡 짧은 데모 영상.
- **전제 조건:** Tier 1로 전환율·리텐션이 검증됐을 것.

## 3. 인디가 실제로 쓰는 "공짜 유입" 전술

| 전술 | 핵심 | 주의 |
|---|---|---|
| **빌드 인 퍼블릭** | 개발 과정·고민·숫자를 X/스레드에 공개. 제품이 아니라 *사람*에 팬이 붙음 → 출시 시 초기 유저로 전환 | 꾸준함이 전부. 출시 전부터 |
| **데모 영상** | "손글씨 쓰니 AI가 바로 첨삭" 같은 15초 화면녹화 1개 > 텍스트 광고 10개. 틱톡/릴스/쇼츠/X 재활용 | 움직임이 본질인 제품일수록 영상이 정답 |
| **커뮤니티 침투** | 타겟이 모인 곳에서 *질문에 답하다가 자연스럽게 언급* | 노골적 자기홍보 글은 차단·역효과. "학습법 공유 → 도구 부차 언급" 포맷 |
| **마이크로 인플루언서** | 1만~5만짜리에게 무료 제공+소액. 메가보다 ROI 우위 | 타겟 정합(우리 유저가 보는 채널)이 팔로워 수보다 중요 |

## 4. 측정 — 안 재면 다 헛수고

인디 최다 실수: 여기저기 뿌리고 **뭐가 먹혔는지 모름**.

- **App Store Connect 무료 분석** 최소 확인: 노출수, 제품페이지 조회→다운로드 **전환율**, 소스 타입(검색/탐색/외부링크/광고).
- **채널별 추적 링크**(앱스토어 Campaign Links): 어느 유입이 실제 설치/잔존으로 이어지는지 구분.
- **핵심 지표는 설치수가 아니라 D1/D7 리텐션.** 안 남으면 광고 늘려봐야 무의미.

---

## 5. 인디 출시 마케팅 타임라인

1. **출시 전** — ASO 키워드 세팅, 데모영상 2~3개, 빌드인퍼블릭으로 청중 모으기, 피칭 리스트 작성. *(마케팅은 출시일이 아니라 그 전에 시작)*
2. **출시 주** — Product Hunt/긱뉴스/커뮤니티 동시 런칭 → 스파이크로 랭킹 부스트.
3. **출시 후 4주** — 콘텐츠 지속 + 앱 내 평점 요청 + 분석으로 전환율 점검.
4. **데이터 확보 후** — 전환 좋은 키워드/오디언스에만 소액 유료 실험.

---

## 6. 전형적 실수 (피해야 할 것)

- 제품 완성 전 마케팅 0 → **출시일에 청중 0명**.
- "좋은 앱이면 알아서 퍼진다" 신화 — **분배(distribution)는 별도 작업**이다.
- 유료광고를 ASO/콘텐츠보다 먼저 태움.
- 리뷰/평점 방치 — 별점은 검색랭킹+전환에 직결.
- 한 번 뿌리고 끝 — 마케팅은 **반복·복리**.

---

## 7. 한 줄 요약

가장 인디스럽고 ROI 높은 셋: **① ASO 키워드 정비 + ② 짧은 데모영상 + ③ 타겟 커뮤니티/마이크로 인플루언서.** 유료광고(ASA 포함)는 이 셋으로 전환이 검증된 *다음*.

## 부록 A — Apple Search Ads (ASA) 요점

유료를 켤 때를 위한 참조. **Tier 3이므로 데이터 확보 후.**

- **Basic vs Advanced**: Basic은 예산만 설정(CPI, 자동), 키워드 데이터 안 보임. **Advanced**(CPT 경매)는 키워드·입찰 직접 운영, 학습 가능 → 인디도 Advanced의 "Search Results" 캠페인부터.
- **노출 슬롯**: Search Results(검색어 1번 슬롯, ROI 최고 — 여기부터) → Search Tab → Today/Product Page(브랜드·스케일업용).
- **운영 루프**: Broad match로 깔기 → Search Terms 리포트에서 전환되는 검색어를 Exact로 승격, 안 되는 건 negative. 이 루프가 ASA의 전부.
- **시작**: `searchads.apple.com` → 개발자 Apple ID 로그인 → 결제·세금 등록 → 국가·앱 선택 → Search Results 캠페인 → 일 예산·기본 입찰가.
- **비용 감각**: 처음 일 $10~20로 1~2주, 키워드 데이터부터. 전환 좋은 키워드에 예산 집중.
- **어트리뷰션**: ASA 자체 제공(Apple Ads Attribution API / AdServices 프레임워크). 콘솔에서 설치 카운트는 기본 확인 가능.
