# Frozen 영역 빨강 주석 (추후 구현 명세)

> 상태: **보류 (reverted)**. 2026-06 세션에서 1차 구현했으나 PencilKit 간섭으로 리버트.
> UX 가치는 높아 추후 재구현 대상으로 남김.

## 목표 / UX

피드백을 이미 받은(=frozen) 영역에 사용자가 필기하면:

1. **자동으로 빨강(주석)으로 표시** — "이건 새 학습 내용이 아니라 받은 피드백에 대한 메모"라는 시각 신호.
2. **다음 피드백 대상에서 제외** — 주석은 채점 대상이 아님.
3. **해당 피드백 되돌리기 시 함께 삭제** — 피드백과 그에 단 메모가 한 묶음으로 사라짐.

멘탈 모델은 단순하다: **"빨강 = 주석"**. 일반 콘텐츠는 frozen 경계 아래에 기본색으로 쓰고, frozen 영역 위에 쓰는 건 전부 주석.

## 소유권 모델 (확정: 세로 band 방식)

피드백 #N이 동결한 구간 = `[이전 frozenBottom, 이번 frozenBottom)`.
되돌리기 시 **그 band 안의 빨강 stroke만 삭제**, 검정 콘텐츠는 보존(동결만 풀려 재요청 대상으로 복귀).
이전 피드백 영역의 빨강은 `minY < 새 frozenBottom`이라 생존.
→ per-stroke 소유권 추적 불필요, 순수 공간 계산으로 성립.

## 1차 구현이 실패한 이유 (핵심 — 재구현 시 반드시 해결)

뿌리는 하나: **PencilKit은 단일 평면 드로잉 + 자체 undo/eraser를 가지며, 우리의 "영역(frozen/주석)" 개념을 전혀 모른다.** 그 위에 영역 시맨틱을 얹으려다 두 가지가 깨졌다.

### 버그 1 — native undo가 빨강 재색칠과 desync
재색칠을 `canvasView.drawing = PKDrawing(strokes: mutated)`로 **드로잉 전체 교체**로 구현했는데, PencilKit의 undo 매니저는 사용자가 그은 액션만 추적한다. 통째 swap은 undo 스택과 어긋나 undo가 빨강 stroke를 못 잡는다.

### 버그 2 — 지우개가 frozen 콘텐츠까지 삭제 (사실 이 기능과 **무관한 선재 버그**)
기존 frozen 보호는 `canvasViewDrawingDidChange`에서 **추가된 stroke**(`count > previousStrokeCount`)만 검사한다. 지우개는 stroke를 **줄이므로** 처음부터 가드된 적이 없다. → 빨강 주석을 빼도 이 버그는 남는다. 별도 과제.

## 권장 아키텍처: 레이어 분리

두 버그의 공통 뿌리("PencilKit이 영역을 모름")를 구조적으로 푸는 방법.

- **콘텐츠 레이어**: frozen된(피드백 완료) 필기. 읽기 전용으로 잠금 → 지우개/필기 입력 비활성(`isUserInteractionEnabled` 또는 별도 비편집 캔버스).
- **활성 레이어**: 신규 필기 + 주석. 자체 PKCanvasView라 독립된 undo/eraser 보유.
- 두 캔버스를 겹쳐(overlay) 시각적으로 한 장처럼 보이게.

이러면:
- undo/eraser가 각 레이어 안에서만 작동 → 버그 1·2 동시 해결.
- frozen 콘텐츠는 물리적으로 다른 캔버스라 지워질 수 없음.
- 주석은 활성 레이어에서 색만 빨강으로(입력 시점에 펜 색 지정 가능 → 재색칠 swap 불필요 → undo 정상).

비용: 드로잉 아키텍처 재작업(저장/로드 시 두 레이어 직렬화, frozen 승격 시 stroke를 콘텐츠 레이어로 이동, 좌표/스크롤 동기화). 작지 않음.

## 재구현 체크리스트

- [ ] 콘텐츠/활성 2-레이어 캔버스 구조
- [ ] frozen 승격: 피드백 확정 시 활성 레이어의 해당 stroke → 콘텐츠 레이어로 이동
- [ ] 활성 레이어에서 주석은 입력 시점에 빨강 펜으로(통째 swap 금지)
- [ ] 피드백 대상 선정: 활성 레이어의 비-주석 stroke만
- [ ] 되돌리기: band 기반으로 콘텐츠 레이어 stroke를 활성 레이어로 되돌리고, band 내 주석 삭제
- [ ] 저장/로드: 두 레이어 + frozenBottom 직렬화
- [ ] 회귀: undo가 주석/콘텐츠 각각에 정상 작동, 지우개가 콘텐츠 레이어를 못 건드림
