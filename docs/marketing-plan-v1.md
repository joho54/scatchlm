## 1. 포지셔닝

### 1.1 타깃 한 문장
> **교재 펴고 손으로 써가며 혼자 깊게 공부하는데, 어려운 부분을 즉시 피드백받고 그 자리에서 반복 연습해 체화하고 싶은 학습자.**

**구체적 순간(메시지의 기준점):**
> *헬라어 격변화를(또는 전공 개념을, 수식을) 손으로 적어보는데 — 이게 맞나? 물어볼 데가 없다.*

### 1.2 타깃 시장 — 좁게
- **타깃:** 진지한 텍스트 기반 학습자 — 고전어(라틴어·헬라어 등)·전공 공부·기술/이론 심화.
- **비타깃(의도적 제외):** 가벼운 회화 학습자(Duolingo 시장). 거기선 우리가 약자 → 진입하지 않는다.
- **근거:** ① 경쟁이 비어 있는 니치(메이저 앱들이 안 건드림) ② 창업자 본인이 실제 가치를 본 영역(Transformer 심화·헬라어·개발 공부 — dogfooding).

### 1.3 차별점 — "닫힌 학습 루프"
강점은 "손글씨 + AI 피드백"이 아니라 그보다 한 단계 위:

> **질문 → 답 받기 → 캔버스에 스크랩 → 그 위에서 손으로 다시 연습 → 스크랩을 컨텍스트로 다시 피드백**

이 사이클 전체가 **한 캔버스에서 닫힌다.** 경쟁 대비:
| | 답변 | 손 연습 | 재피드백 | 한 곳에서 |
|---|---|---|---|---|
| ChatGPT | ✅ | ✗ | ✗ | ✗ |
| 필기 앱(GoodNotes 등) | ✗ | ✅ | ✗ | ✗ |
| **ScatchLM** | ✅ | ✅ | ✅ | ✅ |

학습과학적으로도 능동 재생(active recall) + 반복 피드백 구조라 "한 번 읽고 넘기기"보다 체화에 유리. **"손글씨 AI 튜터"는 모방 가능하지만, 이 루프 전체 통합은 따라하기 어렵다 — 이것이 핵심 마케팅 포인트.**

### 1.4 자산 언어 — KR 몰빵 확정

**배포 지리는 결정 대상이 아니다.** 앱스토어는 기본이 전 세계 동시 배포이고 국가 제한을 걸 이유도 없다 — 출시 순간 누구나 다운로드 가능하며, 제품은 언어 불가지론(Vision·RAG)이라 어느 언어로 깔든 그냥 작동한다. **언어권별 "서비스 작업"은 존재하지 않는다.** 진짜 비용은 단 하나, **마케팅 presence(발품)** 뿐이고 이는 능동 공략하는 곳에만 발생한다.

**결론: 한국 능동 마케팅에 몰빵. 영어는 능동 공략 전까지 미착수.**
- **왜 몰빵:** 솔로의 희소 자원은 창업자 시간 하나. 얕게 여러 시장에 흩뿌리는 것보다 한 곳(에타→국내 어학/전공 커뮤니티, §3)에 모아 **깊이**를 내는 게 좁은 니치의 정석. 메시지·온보딩·가격을 한 시장 피드백으로 빠르게 수렴(=v1 목표).
- **"한국 몰빵" ≠ "한국 전용 앱":** 능동 마케팅만 한국에 한정. 배포는 글로벌 유지(공짜). 비-한국 유저가 검색·입소문으로 들어오면 **한계비용 0으로 서비스**된다 — 끄는 게 아니라 안 미는 것.
- **영어 자산 미제작(출시 범위 제외):** 초기엔 리뷰 0이라 passive 발견이 ≈0 → 영어 자산을 만들 ROI가 없다. 능동 공략을 결정할 때 비로소 착수(아래 트리거). 상세는 `marketing-en-localization-plan.md`(트리거 stub).
- **재정렬 트리거:** §4 지표에서 비-한국어 유입/사용이 의미 있게 잡히거나 글로벌 확장을 의식적으로 결정할 때 → 영어 자산·채널을 미결(§5)에서 승격.
- **데이터 해석 주의:** 외대 특수어 전공자는 "물어볼 데 없는 진지한 텍스트 학습자" 가설의 **좋은 한 조각**이지 전부가 아니다. 여기서 통해도 글로벌(서구권 고전어 등)은 **버린 게 아니라 upside로 남는다** — "이건 한국 제품"으로 결론 점프 금지.

### 2.3 스크린샷 계획 (iPad 세로, 실제 화면 기반)

> **방향 = 세로(portrait).** 앱이 세로 기준이므로 스크린샷도 세로(`2048×2732`)로 통일. App Store는 iPad 세로·가로 둘 다 허용하므로 앱 방향과 일치시킨다. 루프(아래)는 한 장에 욱여넣지 않고 **3장으로 분리**해 흐름을 표현한다 → 총 7장.
> 제작 자산: `appstore-screenshots/`(HTML/CSS + Playwright 렌더, 짙은 파랑 테마). Figma 없이 코드로 관리, 영어판은 텍스트만 교체.

| # | 화면 | 상단 카피 | 목적 |
|---|---|---|---|
| 1 | 손글씨 + AI 피드백 한 화면 | **손으로 쓰면, AI가 바로 첨삭** | 1초에 "뭐가 다른지" |
| 2 | 교재 PDF + 페이지 인용 답변 | **내 교재 기준으로 답해줘요** | grounding 가치 |
| 3 | 루프 ① 질문→AI답변 | **묻고** → 연습하고 → 다시 피드백 | ★ 핵심 차별점(닫힌 루프) |
| 4 | 루프 ② 스크랩 위 손 연습 | 묻고 → **연습하고** → 다시 피드백 | ★ 닫힌 루프 |
| 5 | 루프 ③ 재피드백 | 묻고 → 연습하고 → **다시 피드백** | ★ 닫힌 루프 |
| 6 | 실제 사용 장면(헬라어/수식 등) | **외국어든 전공이든** | 범용성·니치 |
| 7 | 채팅 심화 | **막히면 끝까지 파고들기** | 보조 |

- 3~5번이 "왜 다른가"를 보여주는 핵심 시퀀스. 경쟁 앱은 이 루프를 못 보여준다.
- 목업보다 **실제 손글씨가 들어간 진짜 화면**이 설득력 높음.

**루프 제작 방식 — "흐름(3단계)"이라 정적 1장에 안 담긴다.** 세로에선 3분할/3단 적층이 좁아 손글씨가 안 읽힘. 해결:
- **출시: 3장 분리(#3~5)** — 각 단계를 풀화면 한 장씩. 상단에 `묻고 → 연습하고 → 다시 피드백` **진행 트래커**를 깔고 현재 단계만 강조(흰색) → 3장이 한 흐름으로 묶이고, 넘기는 동작 자체가 "순서"를 전달.
- **fast-follow: 앱 프리뷰 영상(~15초)** — 움직임이 본질인 차별점은 영상이 정답. 출시엔 3장 분리로 가고, 전환율 보며 영상 추가.
- 스크린샷의 일은 "기능 완벽 설명"이 아니라 "다운로드 결정" — **'흐름이 있다'는 호기심만 전달하면 충분**하고, 완성도는 실사용에서 체감된다.

### 2.4 무료 티어 · 가격 (전환 설계)

무료 한도는 마케팅의 **입구(활성화)**다. 첫 세션에 차별점(닫힌 루프)을 무료로 한 바퀴 돌게 하는 게 전환의 전제. 구조는 `iap-subscription-spec.md §4.4` 기준:

- **무료(normal)는 기능 제한이 아니라 일일 사용량 제한.** 전 기능(피드백·교재 grounding·스크랩 루프·채팅)을 쓰되 **하루 사용량 한도**(`DAILY_COST_LIMIT_NORMAL_USD`=$0.15 ≈ 복잡 피드백 7건/일 또는 단순 150건/일)만 둔다. → **"무료로도 루프 전체를 경험"**할 수 있다는 게 마케팅 메시지의 강점. 기능 잠금 방식보다 활성화에 유리.
- **Pro(₩9,900/월)는 "기능 해제"가 아니라 "넉넉한 일일량".** 따라서 카피는 *"X 기능을 쓰려면 결제"*가 아니라 *"매일 깊게 공부할 만큼 충분한 사용량"*으로 간다(현행 §2.2 설명과 일치). Pro 한도($1.00/일 ≈ 복잡 50건)는 가격 산정이 아니라 abuse 상한.
- **활성화 마찰 1 — 교재 없음:** "내 교재 기준"(§2.2·스크린샷 #2)은 PDF 업로드 전제인데 신규 유저는 교재가 없다. → **샘플 교재 1~2개 번들/온보딩 제공**을 활성화 자산으로 다룬다(헬라어 문법서·Transformer 자료 등). 약속한 가치를 첫 세션에 재현하지 못하면 #2 스크린샷이 공수표가 된다.
- **활성화 마찰 2 — 무료 한도 체감:** 무료 7건/일이 "루프 한 바퀴(질문→스크랩→재연습→재피드백)"를 막지 않는지 베타 데이터로 확인(루프 1회가 한도를 다 먹으면 활성화 실패). §4 "루프 사용" 지표와 연계.
- **paywall 노출 타이밍:** 무료 한도 도달(429) 시 업그레이드 CTA(`PaywallView`, IAP 스펙 B-3). **루프를 한 번 완주해 가치를 본 직후**가 전환 골든 타임 — 첫 화면 강제 paywall은 피한다.
- **가격 anchoring(₩9,900):** 타깃 다수가 이미 ChatGPT Plus를 결제 중 → *"그거 있는데 왜 또?"*에 대한 답(손글씨 입력 + 교재 grounding + 한 캔버스 닫힌 루프 = ChatGPT로 안 됨)을 카피·스크린샷에서 한 번 더 명시. 무료체험(intro offer)·연간 플랜은 가격 데이터 후 확장(IAP 스펙 §1 비범위).

## 3. 채널 계획 (단계별)

| 단계 | 매체 | 내용 |
|---|---|---|
| **출시 시점** | 앱스토어 제품 페이지 | 스크린샷 中心(§2). 1차이자 전환의 핵심 |
| **출시 직후 (1차 beachhead)** | **한국외대 에타(Everytime) / 특수어 전공자** | 창업자가 **동문**이라 외부 마케터가 아닌 내부자로 접근 가능. 아래 "왜 이 beachhead인가" 참조 |
| **그 다음** | 좁은 타깃 커뮤니티 | **창업자 사용기**(추상 슬로건보다 강함): Transformer 심화·헬라어 학습 실제 사례. 고전어/전공/기술 학습이 모이는 곳(국내 어학/전공 커뮤니티·대학) |
| **데이터 확보 후** | ASO·채널·유료 | 키워드 튜닝, 전환율 기반 메시지 최적화, 유료 광고 |

**원칙:** 출시 초기 마케팅 = 정교한 광고 카피가 아니라 **구체적 사용기 1~2개**. 0→1 구간의 유통은 사실상 발품이며 그게 정석(다른 채널은 0에선 작동 안 함 — ASO는 리뷰·랭킹 0, 유료는 전환 데이터 0). **"마케터가 되라"가 아니라 "내 사람들이 모인 1~2곳에 솔직한 사용기 1~2개".**

### 3.1 왜 외대 에타 / 특수어 전공자가 1차 beachhead인가
- **창업자 적합성:** 동문 → 내부자 포맷("동문인데 우리 같은 사람 쓰라고 만들었다")이 통함. 에타는 홍보에 알레르기가 있어 이 진정성이 거의 유일한 통로.
- **타깃 정합:** 특수어 전공자(아랍어·페르시아어·태국어·힌디어·헝가리어 등)는 §1.1의 *"손으로 적어보는데 — 물어볼 데가 없다"* 가 **일상**. 튜터·자료·Duolingo가 가장 빈약한 집단 → 페인 최대.
- **손글씨 입력이 진짜 강점:** 아랍/태국/데바나가리/키릴 등 **타이핑이 어색한 문자** → 손쓰기가 gimmick이 아니라 가장 자연스러운 입력. ChatGPT 대비 빈틈이 여기서 최대.
- **활성화 마찰 낮음:** 전공자는 교재 PDF를 이미 보유 → "내 교재 연결"(§2.4) 마찰 ↓.
- 단일 학교라 천장은 낮으나 **첫 코호트 데이터엔 국내 최적 교두보**(외대가 특수어 전공 밀도 압도적). 시장 전체가 아니라 beachhead로 사용.

### 3.2 부차 사항
- **에타 규칙:** 자유게시판 vs 홍보게시판 구분, 노골적 광고는 신고·다운보트. "학습법 공유 → 도구 부차 언급" 포맷.
- **어트리뷰션:** "에타 보고 왔다"를 식별할 방법(프로모 코드/안내) — 이 채널 효과 측정용(§4 전환 지표와 연계).
