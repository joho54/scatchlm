# ScatchLM 1차 마케팅 계획서 (출시용 기준안)

> **Status:** v1 (굳힘)
> **Date:** 2026-06-02
> **Scope:** 첫 출시 시점의 포지셔닝·앱스토어 자산·채널·관찰 지표. 데이터 확보 전 단계.

---

## 1. 포지셔닝

### 1.1 타깃 한 문장
> **교재 펴고 손으로 써가며 혼자 깊게 공부하는데, 어려운 부분을 즉시 피드백받고 그 자리에서 반복 연습해 체화하고 싶은 학습자.**

**구체적 순간(메시지의 기준점):**
> *헬라어 격변화를(또는 전공 개념을, 수식을) 손으로 적어보는데 — 이게 맞나? 물어볼 데가 없다.*

### 1.2 타깃 시장 — 좁게
- **타깃:** 진지한 텍스트 기반 학습자 — 고전어(라틴어·헬라어 등)·전공 공부·기술/이론 심화.
- **비타깃(의도적 제외):** 가벼운 회화 학습자(Duolingo 시장). 거기선 우리가 약자 → 진입하지 않는다.
- **근거:** ① 경쟁이 비어 있는 니치(메이저 앱들이 안 건드림) ② 창업자 본인이 실제 가치를 본 영역(Transformer 심화·헬라어·개발 공부 — dogfooding).

### 1.3 차별점 — "닫힌 학습 루프"
강점은 "손글씨 + AI 피드백"이 아니라 그보다 한 단계 위:

> **질문 → 답 받기 → 캔버스에 스크랩 → 그 위에서 손으로 다시 연습 → 스크랩을 컨텍스트로 다시 피드백**

이 사이클 전체가 **한 캔버스에서 닫힌다.** 경쟁 대비:
| | 답변 | 손 연습 | 재피드백 | 한 곳에서 |
|---|---|---|---|---|
| ChatGPT | ✅ | ✗ | ✗ | ✗ |
| 필기 앱(GoodNotes 등) | ✗ | ✅ | ✗ | ✗ |
| **ScatchLM** | ✅ | ✅ | ✅ | ✅ |

학습과학적으로도 능동 재생(active recall) + 반복 피드백 구조라 "한 번 읽고 넘기기"보다 체화에 유리. **"손글씨 AI 튜터"는 모방 가능하지만, 이 루프 전체 통합은 따라하기 어렵다 — 이것이 핵심 마케팅 포인트.**

## 2. 앱스토어 자산 (1차 매체)

우선순위: **스크린샷 > 부제 > (프리뷰 영상) > 설명 > 키워드**. 대부분 유저는 스크린샷만 보고 결정한다.

### 2.1 이름 / 부제
- **이름:** ScatchLM
- **부제(subtitle, ≤30자):** `손으로 쓰면, AI가 바로 첨삭`

### 2.2 설명 전문 (확정)

### 2.3 스크린샷 계획 (iPad 가로, 실제 화면 기반)
| # | 화면 | 상단 카피 | 목적 |
|---|---|---|---|
| 1 | 손글씨 + AI 피드백 한 화면 | **손으로 쓰면, AI가 바로 첨삭** | 1초에 "뭐가 다른지" |
| 2 | 교재 PDF + 페이지 인용 답변 | **내 교재 기준으로 답해줘요** | grounding 가치 |
| 3 | **스크랩 위에서 다시 연습 + 재피드백** | **묻고 → 연습하고 → 다시 피드백** | ★ 핵심 차별점(닫힌 루프) |
| 4 | 실제 사용 장면(헬라어/수식 등) | **외국어든 전공이든** | 범용성·니치 |
| 5 | 채팅 심화 | **막히면 끝까지 파고들기** | 보조 |

- 3번이 "왜 다른가"를 한 장에 담는 핵심 슬라이드. 경쟁 앱은 이 루프를 못 보여준다.
- 목업보다 **실제 손글씨가 들어간 진짜 화면**이 설득력 높음.

## 3. 채널 계획 (단계별)

| 단계 | 매체 | 내용 |
|---|---|---|
| **출시 시점** | 앱스토어 제품 페이지 | 스크린샷 中心(§2). 1차이자 전환의 핵심 |
| **출시 직후** | 좁은 타깃 커뮤니티 | **창업자 사용기**(추상 슬로건보다 강함): Transformer 심화·헬라어 학습 실제 사례. 고전어/전공/기술 학습이 모이는 곳(서브레딧·디스코드·어학/전공 커뮤니티·대학) |
| **데이터 확보 후** | ASO·채널·유료 | 키워드 튜닝, 전환율 기반 메시지 최적화, 유료 광고 |

**원칙:** 출시 초기 마케팅 = 정교한 광고 카피가 아니라 **구체적 사용기 1~2개**. 창업자가 이미 보유(Transformer·헬라어·개발 공부).

## 4. 관찰 지표 (출시 후 — 가설 검증)

포지셔닝 가설("진지한 텍스트 학습자가 더 잘 retain한다")을 데이터로 확인. 기존 적재 데이터 활용:
- **용도 분포:** `LLMUsage.language`·`task_type`, 교재 컨텍스트 사용률, `AIResponse` 내용 → "외국어 vs 비-언어" 비중. 비-언어 비중이 크면 "학습 캔버스" 피벗 신호.
- **리텐션:** 어떤 용도/타깃이 더 오래 쓰는지.
- **전환:** 무료→Pro 전환율, 전환 유저의 사용 패턴(§10.5 SQL과 연계).
- **루프 사용:** 스크랩→재연습→재피드백을 실제로 도는 유저 비중(차별점이 실사용되는가).
